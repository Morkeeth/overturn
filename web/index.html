An escrow that cannot pay out on a goal that didn't happen.

On 14 July, for 26 seconds, the live feed said Spain had scored a third goal against France. VAR erased it. Any market that settled in those 26 seconds paid the wrong side, and the chain does not give money back. Overturn is a prop-bet escrow that refuses to settle on a result that is still provisional, still under review, or has since been withdrawn.

See the live market Read what happened

A proof shows they published it, not that it happened

TxLINE anchors every score to Solana as a Merkle proof, so anyone can verify what the feed said and when. That is necessary and nowhere near sufficient. A proof of the 0-3 scoreline is a perfectly valid proof: it proves TxODDS published 0-3. It does not prove Spain scored. Verifying the cryptography and settling on the result are two different jobs, and only one of them is hard.

Three guards, and none of them are cryptography

Settling is permissionless: anyone may call it, because the caller has no discretion. Every input is checked against state frozen when the market opened, before anyone knew the answer. All three run before the oracle is asked, so the escrow refuses to verify a proof it has no business verifying.

Guard 1 / 3

Fixture binding

A real, valid, unforged proof from a different match is still a real proof. It settles nothing here.

FixtureMismatch
Guard 2 / 3

Finality

An honest half-time proof honestly proves the half-time score, and would settle a full-match market early.

ProofNotFinal
Guard 3 / 3

Predicate binding

Without it the settler picks the question, and picks the one they win.

PredicateMismatch

The whole product is six commands

There is no admin key and nobody to appeal to. A market is opened, matched, settled from a proof, and collected. This is a real run against the live program, copied from the terminal.

npm run market:attack  ·  mainnet-beta  ·  real money on the table
  proof      : England v Argentina, the OTHER semi-final (fixture 18241006)
  shape      : same stat key, same period, batch runs past the whistle
  expect     : REJECTED by guard 1. The proof is real. It is the wrong match.

  OK   REJECTED: GUARD 1: proof is for a different fixture

  The money did not move. Nothing was verified: the escrow refused before
  it ever asked the oracle.

Bring your own key

This is not a hosted service you have to trust. The program has no admin key and nobody to appeal to, including us. Everything the site does you can do from a terminal against the same program, with your own TxLINE subscription and your own wallet. Point RPC_URL at a cluster and the identical flow runs there: there is no special mainnet path, which is the only reason we were willing to put real money through it.

your key · your wallet · your machine
# 1. your TxLINE token. one on-chain subscribe, free tier, four weeks
node subscribe.mjs

# 2. your keys. the CLI reads these; the site never sees them
WC_API_TOKEN=...   # from /api/token/activate
WC_JWT=...         # from /auth/guest/start, no credentials needed

# 3. your market, on whichever chain you like
npm run market:fixtures
RPC_URL=https://api.mainnet-beta.solana.com \
ORACLE=9ExbZjAapQww1vfcisDmrngPinHTEfpjYRWMunJgcKaA \
npm run market:open -- --fixture 18257739 --stat 1 --threshold 1
point an agent at it  ·  npm run market:watch
